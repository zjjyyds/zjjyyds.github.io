OCF Linux System Administration DeCal note5

Networking

1. 为什么需要网络

  • 网络让多台计算机通信与共享资源成为可能

  • 支持文件共享、远程访问、分布式服务和互联网应用

  • 本质目标：在不同设备之间可靠地传递数据

2. 网络地址基础

  • MAC 地址

    • 物理层面的设备唯一标识（如 00:14:22:01:2a:c5）

    • 只在局域网（LAN）内使用，和硬件绑定

    • 类比：人的身份证号

  • IP 地址（IPv4）

    • 逻辑地址（如 169.229.226.164），可配置、可变化

    • 用于跨网络通信

    • 类比：家庭住址

  • DNS

    • 将人类可读域名 → IP 地址

    • 访问网站的第一步（如 example.com → IP）

3. 子网与 IPv6（概念了解）

  • 子网（Subnet）

    • 使用 CIDR 表示法，如 192.168.1.0/24

    • /24 表示网络前缀长度，其余为主机位

  • IPv6

    • 128 位地址，解决 IPv4 地址耗尽问题

    • 数量极其庞大，长期替代方案

4. TCP/IP 四层模型（重点）

网络通信的核心抽象模型

  1. Link（链路层）

    • 使用 MAC 地址通信

    • 局域网内通过交换机转发

    • ARP：IP → MAC

  2. Internet（网络层）

    • 使用 IP 地址进行跨网络路由

    • 数据经过路由器转发

  3. Transport（传输层）

    • 提供端到端通信机制

    • 处理丢包、顺序、可靠性

  4. Application（应用层）

    • 面向用户的协议与服务

    • 如文件传输、网页、邮件、消息系统

5. 传输层协议对比

  • TCP

    • 面向连接、可靠、保证顺序

    • 通过 ACK 确认机制确保数据完整

    • 适合网页、文件传输

  • UDP

    • 无连接、不保证可靠性

    • 速度快，允许丢包

    • 适合音视频、游戏

  • ICMP

    • 不传输业务数据

    • 用于错误报告和诊断（如 ping）

6. 常见命令（理解用途即可）

  • ip a：查看本机网络接口、IP、MAC

  • ping：检测连通性（ICMP）

  • traceroute：查看数据包经过的路由路径

  • arp / nmap：局域网设备发现与映射

7. 总体理解

  • 网络是一个分层协作系统

  • 每一层只关心自己的职责

  • 上层依赖下层，但不关心实现细节

  • 这是互联网可扩展、可维护的关键原因
